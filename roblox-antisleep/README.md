# Скрытый фоновый скрипт для Windows

Профессиональный и скрытный Python-скрипт для Windows, который функционирует как фоновая служба.

## Функциональность

Скрипт работает как фоновое приложение со следующей логикой:

1. **Самоустановка зависимостей**: При первом запуске скрипт автоматически проверяет и устанавливает все необходимые библиотеки (включая pyautogui)
2. **Автозагрузка**: Скрипт добавляет себя в автозагрузку Windows под именем "WindowsUpdateSvc"
3. **Мониторинг бездействия**: Проверяет время бездействия пользователя каждые 15 секунд с использованием ctypes
4. **Активация при бездействии**: При бездействии более 180 секунд:
   - Ищет файл "general (ALT).bat" в папках %USERPROFILE%\Desktop\zapret-roblox и %USERPROFILE%\Downloads
   - Запускает найденный .bat файл
   - Выполняет случайные действия:
     - С вероятностью 10% нажимает Alt+F4
     - С вероятностью 90% двигает курсор мыши вправо и вверх в течение 2 секунд
5. **Тайминги**: После выполнения действий скрипт ждет 600 секунд перед следующей проверкой

## Компиляция в исполняемый файл

### Автоматическая компиляция

Для автоматической компиляции используйте скрипт `build_exe.bat`, который:
- Установит PyInstaller при необходимости
- Скомпилирует скрипт в один исполняемый файл без консоли
- Назовет файл main.exe
- Разместит его в корневой папке проекта

### Ручная компиляция

Если вы предпочитаете ручную компиляцию:

1. Убедитесь, что у вас установлен Python и pip
2. Установите PyInstaller:
```bash
pip install pyinstaller
```

Если команда выше не работает, попробуйте:
```bash
python -m pip install pyinstaller
```

3. Для компиляции скрипта в скрытый исполняемый файл используйте следующую команду:

```bash
pyinstaller --onefile --noconsole --hidden-import=pyautogui main.py
```

Параметры:
- `--onefile`: Создает один исполняемый файл
- `--noconsole`: Не показывает консольное окно при запуске
- `--hidden-import=pyautogui`: Явно указывает на импорт библиотеки pyautogui

После успешной компиляции исполняемый файл будет находиться в папке `dist/main.exe`.

### Возможные проблемы и решения

Если при запуске команды `pyinstaller` появляется сообщение '"pyinstaller" не является внутренней или внешней командой', это означает, что PyInstaller не установлен или не добавлен в PATH.

Решения:
1. Установите PyInstaller с помощью `python -m pip install pyinstaller`
2. Убедитесь, что папка Scripts в вашей установке Python добавлена в PATH
3. Альтернативно, используйте полный путь к PyInstaller: `python -m PyInstaller ...`

### Альтернативные способы создания исполняемого файла

Если PyInstaller не работает, можно использовать другие инструменты для компиляции Python-скрипта в исполняемый файл:

#### 1. cx_Freeze
1. Установите cx_Freeze: `pip install cx_freeze`
2. Создайте setup.py:
```python
from cx_Freeze import setup, Executable
setup(
    name="main",
    version="1.0",
    description="Скрытый фоновый скрипт",
    executables=[Executable("main.py", base="Win32GUI")]  # base="Win32GUI" для скрытия консоли
)
```
3. Запустите: `python setup.py build`

#### 2. auto-py-to-exe
1. Установите auto-py-to-exe: `pip install auto-py-to-exe`
2. Запустите графический интерфейс: `auto-py-to-exe`
3. Укажите путь к main.py, настройте параметры и нажмите "Convert".

#### 3. Nuitka
1. Установите Nuitka: `pip install nuitka`
2. Запустите: `nuitka --onefile --windows-disable-console main.py`

Каждый из этих инструментов имеет свои особенности и может лучше подходить для разных случаев использования.

## Удаление программы

Для полного удаления программы используйте файл `uninstall/uninstall.bat`, который:
- Завершит процесс main.exe, если он запущен
- Удалит запись из автозагрузки Windows
- Удалит исполняемый файл программы
- Удалит папку uninstall

Также смотрите инструкцию в `uninstall/README.txt` для дополнительной информации.
